# Данные для анализа (пара SPARQL + CSV)

В этой папке хранятся **ваши данные**, которые вы будете анализировать в Python-ноутбуках.

## Как сдаём данные (обязательный формат)

Каждый набор данных должен быть представлен **двумя файлами с одинаковым именем**:

- `<name>.sparql` — SPARQL-запрос к Викиданным (Wikidata).
- `<name>.csv` — результат выполнения этого запроса (таблица CSV), выгруженная из сервиса Wikidata Query Service:
  https://query.wikidata.org/

Пример (смотрите `data/examples/`):
- `cartoons_capital_cost.sparql` → `cartoons_capital_cost.csv`
- `cartoons_genre_country_duration.sparql` → `cartoons_genre_country_duration.csv`

Важно: CSV должен быть **получен именно** из одноимённого `.sparql`‑запроса.

## Откуда берём данные

Данные берём **только из Викиданных (Wikidata)**. [file:88]

## Что нужно выбрать перед запросом

Перед тем как писать запрос, определитесь:

- Какие **объекты** Викиданных вы извлекаете (класс/тип/категория; например: фильмы, города, университеты, компании, спортсмены).
- Какие **свойства** этих объектов вы хотите получить (обычно 3–7 свойств): название, страна, дата, жанр, координаты, длительность, бюджет и т.д.

Согласование с преподавателем **приветствуется**, но не является обязательным.

## Требование к “нетривиальности” данных

Выбирайте такие объекты и свойства, чтобы по таблице можно было сделать **осмысленный анализ и визуализацию**:
- сравнения (по странам/жанрам/периодам),
- распределения (гистограммы),
- зависимости (scatter: X против Y),
- несколько серий на одном графике.

Таблица не должна быть “скучной”: например, **только названия + один столбец без идеи анализа** — плохой вариант.

## Уникальность тем в группе

Если несколько студентов выбрали **одинаковые объекты и одинаковый набор свойств**, исследование сдаёт **только один студент** — тот, кто **первым** написал в чат курса, что именно он выбрал (объекты + свойства). Остальные должны выбрать другой объект и/или свойства.

## Требования к файлам

### 1) SPARQL-файл (`.sparql`)
- Файл должен содержать **полный** запрос.
- Рекомендуется в первой строке добавить комментарий, чтобы преподавателю было легко понять смысл запроса.

Рекомендуемый формат первой строки (пример):
`# Тема: ... | Объекты: ... | Свойства: ... | Фильтры: ... | Язык меток: ru | Дата: ...`

Примеры:
- `# Тема: мультфильмы | Объекты: экземпляры Q202866 | Свойства: название, жанр (P136), страна (P495), длительность (P2047), бюджет (P2130) | Язык: ru`
- `# Тема: мультфильмы | Объекты: экземпляры Q202866 | Свойства: название, жанр, страна, длительность, бюджет | Ограничения: только записи, где все свойства заполнены | Язык: ru`

### 2) CSV-файл (`.csv`)
- Формат: CSV, кодировка UTF-8, разделитель — запятая.
- Минимум: **50 строк** и **3 столбца** (если по вашей теме объективно меньше — обсудите с преподавателем заранее).
- CSV — это **выгрузка результата** выполнения соответствующего `.sparql` в https://query.wikidata.org/

## Имена файлов

- Используйте английские буквы, цифры и подчёркивания: `my_topic_01.sparql`, `my_topic_01.csv`.
- Имена в паре должны совпадать полностью (разница только в расширении).
